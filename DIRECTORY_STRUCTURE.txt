Database Tables
===============

patients
├── id (UUID, Primary Key)
├── personal_info (name, DOB, gender, contact)
├── address_info (address, city, state, zip)
├── emergency_contact
├── insurance_info
├── medical_history
└── allergies

appointments
├── id (UUID, Primary Key)
├── patient_id (Foreign Key → patients)
├── date_time
├── type (consultation, follow_up, emergency, surgery, other)
├── status (scheduled, confirmed, in_progress, completed, cancelled, no_show)
├── reason
└── notes

visits
├── id (UUID, Primary Key)
├── patient_id (Foreign Key → patients)
├── appointment_id (Foreign Key → appointments)
├── date_time
├── chief_complaint
├── diagnosis
├── treatment_plan
├── notes
└── follow_up_date

examinations
├── id (UUID, Primary Key)
├── visit_id (Foreign Key → visits)
├── visual_acuity (OD/OS)
├── intraocular_pressure (OD/OS)
├── refraction (OD/OS - sphere, cylinder, axis)
├── pupil_exam (OD/OS)
├── external_exam
├── slit_lamp_exam
├── fundus_exam
└── additional_findings

prescriptions
├── id (UUID, Primary Key)
├── visit_id (Foreign Key → visits)
├── patient_id (Foreign Key → patients)
├── type (eyeglasses, contact_lenses, medication)
├── optical_measurements (sphere, cylinder, axis, add, PD)
├── medication_details (name, dosage, frequency, duration)
├── instructions
├── issued_date
└── expiry_date

bills
├── id (UUID, Primary Key)
├── patient_id (Foreign Key → patients)
├── visit_id (Foreign Key → visits)
├── dates (bill_date, due_date)
├── amounts (total, paid, balance)
├── status (pending, partial, paid, overdue, cancelled)
├── payment_method
└── notes

bill_items
├── id (UUID, Primary Key)
├── bill_id (Foreign Key → bills)
├── description
├── quantity
├── unit_price
└── total_price

payments
├── id (UUID, Primary Key)
├── bill_id (Foreign Key → bills)
├── payment_date
├── amount
├── payment_method
├── reference_number
└── notes


Routes Structure
================

Public Routes:
  /login                    → Login page
  /signup                   → Signup page

Protected Routes (require authentication):
  /                         → Redirect to /dashboard
  /dashboard                → Main dashboard
  /patients                 → Patient list
  /patients/new             → Add new patient
  /patients/:id/edit        → Edit patient
  /appointments             → Appointments
  /visits                   → Visit records
  /billing                  → Billing & invoices
  /settings                 → User settings
  *                         → Redirect to /dashboard

Development Workflow
====================

1. Setup:
   npm install
   cp .env.example .env
   # Edit .env with Supabase credentials
   # Run supabase-schema.sql in Supabase

2. Development:
   npm run dev          # Start dev server
   # Navigate to http://localhost:5173

3. Build:
   npm run build        # Production build
   npm run preview      # Preview build

4. Deploy:
   # Deploy 'dist' folder to hosting platform
   # Set environment variables
